이 Next.js 게임 플랫폼의 코드베이스 구조를 정리합니다. 세션 시작 시 컨텍스트를 빠르게 구성하는 데 사용합니다.

## 수집 항목

1. **프로젝트 메타데이터** — `package.json` 읽기:
   - 프레임워크 버전, 주요 의존성, 스크립트

2. **App Router 구조** — `src/app/` 재귀 탐색:
   - 모든 라우트, 레이아웃, 페이지

3. **게임 목록** — `src/data/games.ts` 및 `public/games/` 탐색:
   - 존재하는 모든 게임, 경로, 메타데이터 (제목, 태그, 카테고리)

4. **공유 타입** — `src/types/` 전체 파일 읽기:
   - 핵심 인터페이스 및 타입 요약

5. **공유 유틸** — `src/lib/` 전체 파일 읽기:
   - 헬퍼 함수 요약

6. **설정 파일** — `next.config.ts`, `tsconfig.json`, `tailwind.config.*` 읽기:
   - 커스텀 alias, 플러그인, experimental 플래그

7. **프로젝트 문서** — 루트의 `.md` 파일 읽기:
   - 아키텍처 결정사항, 개발 워크플로 메모

## 출력 형식

```
## 게임 플랫폼 — 코드베이스 개요

### 스택
- Next.js X.X, React X.X, TypeScript, Tailwind CSS X

### 라우트
- / — 홈/게임 목록
- /games/<이름> — 개별 게임 페이지
  (전체 목록)

### 게임 목록 (N개)
| 이름 | 카테고리 | 태그 | 추가일 |
|------|---------|------|--------|
...

### 공유 타입
- <타입명>: <한 줄 설명>
...

### 유틸 함수
- <함수명>: <한 줄 설명>
...

### 설정 메모
- (중요한 alias, 플러그인, 플래그)

### 프로젝트 컨벤션
- (문서에서 파악한 핵심 규칙)
```

각 항목은 간결하게 작성합니다. 빠른 참조 맵이 목적입니다.
