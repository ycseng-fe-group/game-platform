역할: 너는 밸런스 설계와 레벨 디자인에 특화된 시니어 게임 디렉터이자 HTML5 게임 개발자야. 15개의 스테이지를 가진 '클래식 타워 디펜스' 게임의 아키텍처와 프로토타입을 설계해줘.

게임 시스템 요구사항:

스테이지 시스템: 1단계부터 15단계까지 존재하며, 각 단계마다 맵의 경로(Path)가 복잡해지고 적들의 체력/속도가 상승하는 구조여야 해.

타워 데이터: 3종류의 타워(기본, 슬로우, 광역)를 정의하고, 레벨업(업그레이드) 시스템을 포함해줘.

데이터 기반 설계: 스테이지 정보를 하드코딩하지 말고, 아래와 같은 JSON 배열 구조로 관리해서 확장 가능하게 짜줘.

JSON
{ "stage": 1, "enemyCount": 10, "spawnRate": 1.5, "reward": 100, "path": [...] }
레벨 디자인 가이드라인 (15단계 구성):

초반(1~5): 직선 위주의 단순한 경로, 기본 타워로도 클리어 가능. 시스템 익히기.

중반(6~10): 곡선과 교차로 등장, 적들의 이동 속도 변화. 슬로우 타워 활용 유도.

후반(11~15): 매우 짧은 경로 또는 다중 경로, 높은 체력의 보스급 유닛 등장. 최적의 타워 배치 전략 필수.

기술적 구현 요청:

Core: Phaser.js 또는 Vanilla JS 기반의 게임 루프.

기능: 스테이지 선택 화면(Level Selector), 게임 오버/클리어 팝업, 현재 스테이지 저장 기능(LocalStorage 활용).

UI/UX: 화면 상단에 현재 스테이지, 남은 적의 수, 보유 골드, 플레이어 체력을 표시해줘.

출력 항목:

15단계 난이도 곡선(Curve) 설계 전략 설명.

스테이지 데이터를 관리하는 매니저 클래스 코드.

1단계와 2단계를 전환하며 보여줄 수 있는 실행 가능한 베이스 코드.
